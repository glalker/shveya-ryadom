# Ваша швея рядом

Лёгкий сайт-визитка с приёмом заявок для швеи на дому (ЖК «Парадный ансамбль»,
Пулковское шоссе, СПб). Клиент выбирает услугу, оставляет телефон — заявка
**приходит маме в мессенджер MAX**. Телефон мамы нигде не публикуется: она сама
перезванивает клиенту.

## Что внутри

```
Швея/
├── public/                  ← сам сайт (можно открыть без сервера)
│   ├── index.html
│   ├── css/styles.css
│   └── js/
│       ├── config.js        ← ✏️ ТУТ правим тексты, услуги и цены
│       ├── app.js           ← логика подбора и формы
│       └── maxapp.js        ← задел под мини-апп в MAX
└── server/                  ← бэкенд: принимает заявку и шлёт в MAX
    ├── server.js
    ├── maxClient.js
    ├── package.json
    └── .env.example         ← скопировать в .env и заполнить
```

## Разделы сайта

1. **Подбор + заявка** (как у Aviasales): «что сделать» → «с какой вещью» →
   «какая работа» → срочность → кнопка **Оставить заявку**.
2. **Услуги и цены** — типовой прайс из `config.js`.
3. **Где** — ЖК «Парадный ансамбль», рядом с домом.
4. **Кто я** — опыт 40+ лет, соседка, почему именно ко мне.
5. **История шитья на Руси и в России** — для атмосферы и поисковой выдачи (SEO).
6. **Форма заявки** + пометка о приватности телефона.

## Запуск

Нужен Node.js 18+.

```bash
cd server
cp .env.example .env        # затем впишите токен и ID (см. ниже)
npm install
npm start
```

Откройте `http://localhost:3000`.

> Заявки **всегда** дублируются в файл `server/data/orders.jsonl` (по одной
> JSON-строке на заявку), даже если MAX ещё не настроен — так ничего не потеряется.

## Настройка MAX (куда прилетают заявки)

1. **Создайте бота** в приложении MAX (через системного бота-помощника для
   разработчиков). Получите **токен доступа** — это `MAX_BOT_TOKEN`.
2. **Узнайте ID получателя** (мамы):
   - мама открывает диалог с вашим ботом и пишет ему любое сообщение;
   - выполните запрос обновлений бота (токен передаётся в заголовке `Authorization`):
     ```bash
     curl "https://platform-api2.max.ru/updates" -H "Authorization: ВАШ_ТОКЕН"
     ```
   - в ответе найдите ID отправителя (поле вида `user_id`/`sender.user_id`) —
     это `MAX_RECIPIENT_ID`, тип получателя оставьте `user`.
3. Впишите оба значения в `server/.env`:
   ```
   MAX_BOT_TOKEN=...
   MAX_RECIPIENT_ID=...
   MAX_RECIPIENT_TYPE=user
   ```
4. Перезапустите сервер. Отправьте тестовую заявку с сайта — сообщение придёт маме.

Проверить состояние: `http://localhost:3000/api/health` → `{ "maxConfigured": true }`.

> ⚠️ API MAX развивается. На 2026 год: хост `platform-api2.max.ru`, токен — в
> заголовке `Authorization`, получатель — query-параметр `user_id`/`chat_id`
> (см. https://dev.max.ru/docs-api). Базовый адрес (`MAX_API_BASE`) и тип
> получателя вынесены в `.env`, поэтому при изменениях правится только `.env`,
> без правок кода. Если заявки приходят в группу — поставьте
> `MAX_RECIPIENT_TYPE=chat` и `MAX_RECIPIENT_ID` = ID чата.

## Как править содержимое

Почти всё (тексты, услуги, цены, адрес) — в одном файле **`public/js/config.js`**.
Меняете цену или добавляете услугу — правите массив `categories`. Перезагрузка
страницы сразу показывает изменения. Дизайн (цвета, отступы) — в
`public/css/styles.css`, переменные палитры вверху файла.

## Фотографии

В блоках «Где» и «Кто» стоят рамки-заглушки. Замените их на реальные снимки
(панорамы ЖК «Парадный ансамбль», машинка «Зингер»): добавьте `<img>` вместо
содержимого `.photo-slot` в `index.html` либо задайте фон в `styles.css`.

## Группа и мини-апп в MAX (на будущее)

Сайт уже к этому готов:

- весь фронтенд — статические файлы из `public/`, их можно встроить в мини-апп;
- `public/js/maxapp.js` определяет, открыт ли сайт внутри MAX, и умеет подставлять
  имя/телефон пользователя из MAX в форму (когда появится мост/SDK);
- при отправке заявка помечается источником `max-miniapp` — видно, откуда пришла.

Когда сделаете группу в MAX: создайте мини-апп, укажите адрес этого сайта, а в
`.env` поменяйте `MAX_RECIPIENT_TYPE=chat` и `MAX_RECIPIENT_ID` на ID группы —
тогда заявки будут падать в общий чат.

## Деплой

Подойдёт любой хостинг с Node.js (например, VPS, Render, Railway). Нужно:
держать запущенным `server/`, прокинуть домен на него и задать переменные
окружения из `.env`. Статику `public/` сервер отдаёт сам.
